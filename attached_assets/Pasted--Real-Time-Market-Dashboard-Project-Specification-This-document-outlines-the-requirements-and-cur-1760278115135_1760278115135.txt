This document outlines the requirements and current implementation status for a modular, customizable web dashboard designed for real-time market monitoring and external webhook integration.
I. Project Goal

To deliver a single-file, responsive React application using Tailwind CSS and Firebase Firestore that provides users with a persistent, drag-and-drop dashboard for monitoring live cryptocurrency market data (via WebSocket APIs) and receiving and processing external messages (simulated webhooks).

III. Next Steps (Feature Enhancements)

The following features are required to complete the project vision:

    Market Data Granularity:

        Goal: Enhance the Market Data Widget to display more granular data than just the current price and 24h volume.

        Requirement: Add support for viewing Level 2 Order Book Depth (Top 5 Bids/Asks) in a separate widget or expand the existing one. This requires subscribing to the appropriate WebSocket channel (e.g., depth5 or diff.depth).

    Visualizations:

        Goal: Introduce a chart widget to display historical price movement.

        Requirement: Create a new widget that visualizes K-Line (Candlestick) data using a library like D3.js or a simple SVG rendering to plot the last 100 closing prices received from the market stream.

    Enhanced Notifications (External):

        Goal: Provide a way to push real alerts externally (currently, alerts are only displayed in the Alerts Log widget).

        Requirement (Simulated): Implement a simple visual on-screen "Toast" popup (using React state for notification queuing) whenever a critical alert is triggered, ensuring the user is notified even if they are busy elsewhere on the dashboard.

    Flexible Layout Saving:

        Goal: Improve widget layout flexibility beyond simple vertical reordering.

        Requirement: Replace the custom drag-and-drop move buttons with proper 2D drag-and-drop library integration (e.g., react-grid-layout) to allow resizing and free-form positioning of widgets within the grid.

IV. Technical Constraints & Context

    Platform: React with Functional Components and Hooks.

    Styling: Tailwind CSS.

    Data Persistence: Firebase Firestore (using the existing useFirebase hook and security rules).

    State Management: Local useState and useCallback for performance.

    Single-File Mandate: All code must remain within the single Dashboard.jsx file.

    Exchanges: Binance, Bybit, OKX (Public APIs only).
